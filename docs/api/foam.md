# API Расчёта Пены Роя (Swarm Foam)

Модуль `gra_swarm.foam` реализует математический формализм **Пены Роя** ($\Phi_{swarm}$). В философии GRA пена — это пространство вероятностных переходов между состояниями агентов, где рождается новая информация. Чем выше плотность пены и её турбулентность (разнообразие), тем выше вероятность скачка к новому знанию (инсайту).

## Основная концепция

Пена рассчитывается как функция от распределений состояний агентов в рое. Мы используем обобщенную **KL-дивергенцию** (расстояние Кульбака-Лейблера) для измерения "трения" между индивидуальностями агентов.

$$ \Phi_{swarm} = \frac{1}{N(N-1)} \sum_{i \neq j} D_{KL}(P_i \| P_j) + \lambda \cdot H_{chaos} $$

Где:
*   $P_i$ — распределение вероятностей действий/состояний $i$-го агента.
*   $D_{KL}$ — мера отличия одного агента от другого (источник конфликта и развития).
*   $H_{chaos}$ — энтропийная составляющая, вносимая намеренными ошибками.
*   $\lambda$ — коэффициент "страдания", усиливающий влияние хаоса на структуру пены.

## 1. Класс `SwarmFoamCalculator`

Основной интерфейс для вычисления метрик пены.

### Импорт
```python
from gra_swarm.foam import SwarmFoamCalculator
```

### Конструктор
```python
calculator = SwarmFoamCalculator(
    metric: str = "kl_divergence",
    normalize: bool = True,
    epsilon: float = 1e-10
)
```
*   `metric`: Тип метрики разнообразия (`"kl_divergence"`, `"js_divergence"`, `"wasserstein"`). По умолчанию KL.
*   `normalize`: Если `True`, приводит значение $\Phi_{swarm}$ к диапазону $[0, 1]$.
*   `epsilon`: Малое число для предотвращения деления на ноль при расчете логарифмов (защита от сингулярностей вакуума).

### Методы

#### `calculate(agents: List[GRAAgent]) -> float`
Вычисляет скалярное значение плотности пены для текущего состояния роя.

*   **Аргументы**:
    *   `agents`: Список объектов агентов, имеющих метод `get_distribution()`.
*   **Возвращает**: `float` — значение $\Phi_{swarm}$.
*   **Пример**:
    ```python
    foam_value = calculator.calculate(my_agents)
    print(f"Плотность пены: {foam_value}")
    ```

#### `calculate_matrix(agents: List[GRAAgent]) -> np.ndarray`
Возвращает матрицу попарных дивергенций $M_{ij} = D(A_i, A_j)$. Полезно для визуализации структуры конфликтов в рое.

*   **Возвращает**: `np.ndarray` формы $(N, N)$.
*   **Пример**:
    ```python
    import seaborn as sns
    matrix = calculator.calculate_matrix(my_agents)
    sns.heatmap(matrix) # Визуализация "трения" между агентами
    ```

#### `get_turbulence(agents: List[GRAAgent], history: List[List[GRAAgent]]) -> float`
Вычисляет скорость изменения пены во времени (производная $\frac{d\Phi}{dt}$). Высокая турбулентность указывает на фазу активного обучения или кризиса.

*   **Аргументы**:
    *   `history`: Список снимков состояния роя за предыдущие шаги.
*   **Возвращает**: `float` — индекс турбулентности.

## 2. Модуль `diversity.py` (Низкоуровневые функции)

Здесь реализованы математические ядра для расчета расстояний между распределениями.

### `kl_divergence(p: np.ndarray, q: np.ndarray, epsilon: float) -> float`
Классическая дивергенция Кульбака-Лейблера: $\sum p(x) \log \frac{p(x)}{q(x)}$.
*   **Особенность GRA**: Асимметричность $D_{KL}(p\|q) \neq D_{KL}(q\|p)$ используется для моделирования доминирования одной идеи над другой.

### `js_divergence(p: np.ndarray, q: np.ndarray) -> float`
Симметричная дивергенция Дженсена-Шеннона. Используется, когда важно равноправие агентов в диалоге.

### `entropy_penalty(distributions: List[np.ndarray], lambda_factor: float) -> float`
Добавляет штраф за излишний порядок (низкую энтропию). Если все агенты слишком похожи (консенсус), эта функция резко снижает общую оценку пены, стимулируя введение мутаций.

## 3. Модуль `metrics.py` (Дополнительные метрики)

### `compute_negentropy_flow(foam_history: List[float]) -> float`
Анализирует историю значений пены и вычисляет поток негэнтропии — количество порядка, извлеченного из хаоса за период наблюдения.
*   **Формула**: Интеграл от положительных изменений пены, скорректированный на коэффициент эффективности.

### `detect_phase_transition(foam_values: List[float], threshold: float = 0.8)`
Определяет момент "фазового перехода", когда система переходит от хаотичного поиска к структурированному решению (рождение гениальной идеи).
*   **Возвращает**: `bool` или индекс шага, на котором произошел переход.

## Полный пример использования

```python
from gra_swarm.core.agent import GRAAgent
from gra_swarm.foam.calculator import SwarmFoamCalculator
from gra_swarm.foam.metrics import detect_phase_transition
import numpy as np

# 1. Создаем агентов с разными начальными убеждениями
agents = [
    GRAAgent(f"agent_{i}", state=np.random.dirichlet(np.ones(5))) 
    for i in range(10)
]

# 2. Инициализируем калькулятор пены
foam_calc = SwarmFoamCalculator(metric="kl_divergence", normalize=True)

# 3. Симуляция шага эволюции
current_foam = foam_calc.calculate(agents)
print(f"Текущая плотность пены: {current_foam:.4f}")

# 4. Получаем матрицу конфликтов (кто с кем не согласен)
conflict_matrix = foam_calc.calculate_matrix(agents)
max_conflict_idx = np.unravel_index(np.argmax(conflict_matrix), conflict_matrix.shape)
print(f"Максимальный конфликт между агентами {max_conflict_idx[0]} и {max_conflict_idx[1]}")

# 5. Проверка на фазовый переход (в реальном цикле здесь был бы список истории)

#     print("⚡ Обнаружен прорыв! Система самоорганизовалась.")
```

## Примечания по реализации

1.  **Численная стабильность**: Все функции логарифмирования защищены параметром `epsilon`. В теории GRA "ничто" (вакуум) не может иметь нулевую вероятность, так как это остановило бы генерацию информации.
2.  **Производительность**: Для больших роев ($N > 1000$) рекомендуется использовать аппроксимацию выборки или блочное вычисление матрицы, так как сложность $O(N^2)$.
3.  **Связь с GRA**: Значение $\Phi_{swarm}$ напрямую влияет на шаг обучения в `GRAOptimizer`. Высокая пена увеличивает шаг градиента (агент смелее экспериментирует), низкая — уменьшает (агент шлифует детали).
