# API Оптимизатора (GRA Optimizer)

Модуль `gra_swarm.optimizer` содержит реализацию **GRA-оптимизатора** (`GRAOptimizer`) — алгоритма, управляющего эволюцией роя агентов. В отличие от традиционных оптимизаторов (SGD, Adam), которые стремятся минимизировать функцию потерь любым путем, GRA-оптимизатор намеренно вводит контролируемый хаос (ошибки) и использует «страдание» агентов как движущую силу для выхода из локальных минимумов.

## Философия алгоритма

Оптимизация в GRA-Swarm рассматривается как борьба между двумя силами:
1.  **Энтропия ($\sigma$)**: Случайные мутации, ошибки, дестабилизация. Необходима для исследования пространства решений (Vacuum $\to$ Information).
2.  **Негэнтропия ($\eta$)**: Стремление к порядку, градиентный спуск, консолидация знаний.
3.  **Страдание ($\lambda$)**: Функция веса ошибки. Агенты с высоким $\lambda$ сильнее реагируют на неудачи, что ускоряет их адаптацию, но повышает риск «сгорания» (дивергенции).

Цель оптимизатора — не просто найти минимум функции $L(x)$, а провести систему через фазовый переход, где рождается новое качество решения (Гениальность).

$$ \Delta x_t = -\eta \cdot \nabla L(x_t) + \sigma \cdot \mathcal{N}(0, 1) \cdot (1 + \lambda \cdot L(x_t)) $$

Где второй член уравнения показывает, что величина случайной ошибки растет пропорционально текущему уровню «страдания» (значению функции потерь).

## 1. Класс `GRAOptimizer`

Основной двигатель эволюции роя.

### Импорт
```python
from gra_swarm.optimizer import GRAOptimizer
```

### Конструктор
```python
optimizer = GRAOptimizer(
    agents: List[GRAAgent],
    objective: Callable,
    learning_rate: float = 0.01,
    entropy_factor: float = 0.1,
    suffering_amplifier: float = 1.0,
    foam_coupling: bool = True,
    device: str = "cpu"
)
```

**Параметры:**
*   `agents`: Список объектов `GRAAgent`, подлежащих оптимизации.
*   `objective`: Целевая функция $L(x)$, которую нужно минимизировать (или максимизировать).
*   `learning_rate` ($\eta$): Базовая скорость движения к порядку (градиентный шаг).
*   `entropy_factor` ($\sigma$): Базовый уровень шума/ошибок.
*   `suffering_amplifier` ($\lambda$): Коэффициент усиления ошибок при высоких потерях. Чем выше, тем агрессивнее агент экспериментирует, когда ему «больно».
*   `foam_coupling`: Если `True`, параметры $\sigma$ и $\eta$ динамически调整руются в зависимости от плотности пены $\Phi_{swarm}$ (см. [API Foam](foam.md)).
    *   *Высокая пена* $\to$ уменьшаем шум (консолидация).
    *   *Низкая пена* (застой) $\to$ увеличиваем шум (прорыв).
*   `device`: Устройство для вычисций (`cpu`, `cuda`).

### Методы

#### `step() -> Dict[str, float]`
Выполняет один шаг эволюции для всех агентов.
1.  Вычисляет градиенты (или аппроксимации) для каждого агента.
2.  Рассчитывает индивидуальный уровень «страдания» на основе текущей потери.
3.  Генерирует стохастическую ошибку, масштабированную по $\lambda$.
4.  Обновляет состояния агентов.
5.  (Опционально) Пересчитывает метрики пены для коррекции параметров.

*   **Возвращает**: Словарь со статистикой шага:
    ```python
    {
        "mean_loss": 0.45,
        "max_suffering": 0.92,
        "foam_density": 1.23,
        "entropy_injected": 0.05
    }
    ```

#### `run(steps: int, verbose: bool = False, callback: Callable = None) -> History`
Запускает полный цикл оптимизации.

*   **Аргументы**:
    *   `steps`: Количество итераций.
    *   `verbose`: Выводить ли логи о «страданиях» и прорывах агентов.
    *   `callback`: Функция, вызываемая после каждого шага (например, для логирования или визуализации).
*   **Возвращает**: Объект `History`, содержащий траектории потерь, изменения параметров и моменты фазовых переходов.

#### `reset_states()`
Сбрасывает внутренние состояния оптимизатора (моментумы, историю пены), но сохраняет веса агентов. Полезно при смене задачи.

#### `get_best_agent() -> GRAAgent`
Возвращает агента с наименьшей текущей потерей. Это кандидат в «Гении» текущего цикла.

## 2. Модуль `scheduler.py` (Планировщик)

Реализует стратегии изменения параметров во времени. В GRA важно не просто уменьшить шум до нуля, а провести его через пики в критические моменты.

### Класс `SufferingScheduler`

Управляет динамикой коэффициента $\lambda$ (страдания).

```python
from gra_swarm.optimizer.scheduler import SufferingScheduler

scheduler = SufferingScheduler(strategy="cyclic_suffering")
```

**Доступные стратегии:**
1.  `"linear_decay"`: Классическое затухание шума. Подходит для простых задач.
2.  `"cyclic_suffering"`: Периодическое увеличение уровня ошибок. Имитирует кризисы развития, необходимые для скачка сложности.
3.  `"adaptive_pain"`: $\lambda$ увеличивается автоматически, если потеря не уменьшается в течение $N$ шагов (агент «застрял» и должен «пострадать», чтобы выйти).
4.  `"chaos_birth"`: Начинается с высокого хаоса, резко падает при обнаружении перспективной области, затем снова растет для уточнения границ.

### Метод `update(step: int, current_loss: float, foam: float) -> Tuple[float, float]`
Возвращает новые значения `learning_rate` и `entropy_factor` для текущего шага.

## Полный пример использования

Этот пример демонстрирует создание оптимизатора, который ведет агентов через «долину страданий» к глобальному минимуму сложной многоэкстремальной функции.

```python
import numpy as np
from gra_swarm.core.agent import GRAAgent
from gra_swarm.core.personality import PersonalityGenerator
from gra_swarm.optimizer import GRAOptimizer
from gra_swarm.optimizer.scheduler import SufferingScheduler

# 1. Определение сложной целевой функции (Ландшафт с множеством локальных минимумов)
def complex_landscape(x):
    # Комбинация сферы и высокочастотного шума (имитация хаоса реальности)
    return np.sum(x**2) + 0.5 * np.sin(10 * np.sum(x)) + 0.2 * np.random.randn()

# 2. Создание роя агентов с разными профилями
pg = PersonalityGenerator(seed=42)
agents = []
for i in range(20):
    # Создаем агентов: некоторые "трусливые" (низкий lambda), некоторые "герои" (высокий lambda)
    profile = pg.generate_profile(type="random", lambda_range=(0.1, 0.9))
    agent = GRAAgent(
        agent_id=f"Seeker-{i}",
        personality=profile,
        initial_state=np.random.uniform(-5, 5, size=2) # 2D пространство поиска
    )
    agents.append(agent)

# 3. Инициализация оптимизатора
optimizer = GRAOptimizer(
    agents=agents,
    objective=complex_landscape,
    learning_rate=0.05,
    entropy_factor=0.2,       # Высокий начальный хаос
    suffering_amplifier=1.5,  # Ошибки сильно усиливаются при больших потерях
    foam_coupling=True        # Включить связь с плотностью пены
)

# 4. Настройка планировщика страданий
scheduler = SufferingScheduler(strategy="cyclic_suffering")

print("🚀 Запуск эволюции через страдание...")

# 5. Цикл обучения
history_log = []
for step in range(100):
    # Получаем новые параметры от планировщика
    lr, entropy = scheduler.update(step, optimizer.get_current_loss(), optimizer.get_foam())
    optimizer.learning_rate = lr
    optimizer.entropy_factor = entropy
    
    # Шаг эволюции
    stats = optimizer.step()
    
    history_log.append(stats["mean_loss"])
    
    if step % 10 == 0:
        best = optimizer.get_best_agent()
        print(f"Step {step}: Loss={stats['mean_loss']:.4f}, Foam={stats['foam_density']:.2f}, "
              f"Best Agent: {best.id} (Suffering: {best.current_loss:.2f})")

# 6. Анализ результата
final_best = optimizer.get_best_agent()
print(f"\n🏆 Эволюция завершена.")
print(f"Лучшее решение найдено агентом {final_best.id} с потерей {final_best.current_loss:.6f}")
print(f"Агент прошел через {final_best.total_critical_errors} критических ошибок.")
```

## Особенности реализации для GRA-Swarm

1.  **Индивидуальный подход**: В отличие от стандартных оптимизаторов, где параметры глобальны, здесь каждый агент обновляется согласно своей личности ($P_i$). Агент с высокой негэнтропией будет игнорировать часть шума, в то время как хаотичный агент будет использовать его для прыжков.
2.  **Обратная связь от Пены**: Если `foam_coupling=True`, оптимизатор мониторит разнообразие роя. Если агенты слишком похожи (консенсус, застой), оптимизатор искусственно увеличивает $\sigma$ (энтропию), вызывая «кризис», чтобы разрушить стагнацию.
3.  **Память ошибок**: Агенты запоминают свои худшие состояния. При повторном попадании в похожую область пространство решений искажается (штраф), заставляя рой искать новые пути. Это реализация принципа «учиться на ошибках».
